梯子安装说明

Written by yyc on October 18, 2019 in Linux

Linux发行版选择

现在一般的vps服务商主要提供以下几种Linux发型版：CentOS、Ubuntu、Debian，我个人一般使用Centos来用，虽然这种确实不好用，不如Debian和Ubuntu操作方便，但是用习惯了也就忍了。

bbr加速

大部分人购买vps的主要用途是搭建梯子，那么网速就是头等大事，在无法改变带宽的情况下，通过提高网络吞吐量来加速网络是一种切实可行的方法。

bbr是谷歌研发的一种TCP拥塞控制算法，它可以有效提高吞吐量和降低网络延迟，并且Linux Kernel4.9+已经集成了该算法。

先检查Linux内核版本：

# uname -r

看看结果，如果是输出类似：

3.10.0-957.5.1.el7.x86_64

则表示先升级内核，然后才能开启Google BBR。（升级内核暂时跳过，可自行谷歌）

若内核大于4.9，那么我们直接使用秋水逸冰的一键脚本即可：

wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh
chmod +x bbr.sh
./bbr.sh

完成后检查一下结果，看是不是确实成功开启：

sysctl net.ipv4.tcp_congestion_control

如果成功，返回值一般为net.ipv4.tcp_congestion_control = bbr

V2ray的安装和使用

众所周知，ss/ssr已经被墙完美识别，那么现存的方法其实主要就是v2ray和wireguard等富强技术。wireguard还不是那么成熟和稳定，因此我自己选用v2ray。

首先，使用一键脚本安装v2ray。

wget https://install.direct/go.sh
bash go.sh

安装好之后，配一下自带的配置文件，地址在/etc/v2ray/config.json。

如果仅仅是简单实用google等，偶尔查阅使用，可以简单配置下，使用tcp代理。

{
  "inbounds": [{
    "port": 选择一个端口号,
    "protocol": "vmess",
    "settings": {
      "clients": [
        {
          "id": "选择一个ID",
          "level": 1,
          "alterId": 64
        }
      ]
    }
  }],
  "outbounds": [{
    "protocol": "freedom",
    "settings": {}
  },{
    "protocol": "blackhole",
    "settings": {},
    "tag": "blocked"
  }],
  "routing": {
    "rules": [
      {
        "type": "field",
        "ip": ["geoip:private"],
        "outboundTag": "blocked"
      }
    ]
  }
}

如果是深度用户，那么可能使用WebSocket+TLS+CDN的方式更为可靠，因为如果富强被发现的话，IP可能会被封，谁都不希望自己花钱买的vps不能用，所以，最好还是用这种方法更保险，按照以下步骤来：

  1. 购买域名并添加A记录解析到你的VPS主机IP地址；（如果是国外的域名不需要实名认证，如果是国内的域名，需要实名认证，如果想打开80端口，那么还需要备案，但是这样也有好处，可以套个国内的CDN，加速效果更好）；
  2. 使用Caddy/Nginx等软件反代，这里我们使用Caddy，因为它是使用golang编写的，部署和使用更为便捷（特别是https）；
  3. 套CDN，如果是国外的域名，那使用CloudFlare可能是一个比较好的选择。

Caddy的安装和使用

使用官方提供的脚本直接安装：

curl -s https://getcaddy.com | bash

安装完了可以用which caddy命令查看一下安装位置，一般情况下是在/usr/local/bin/caddy

在/etc下创建caddy的配置文件/etc/caddy/caddy.config：

****.com   //此处填写你的域名
{
  log ./caddy.log
  proxy /ray localhost:端口号 {
    websocket
    header_upstream -Origin
  }
}

运行caddy：

caddy -conf /etc/caddy/caddy.config
